# Simulation Guidance for Story Generation

## Before Writing a Scene

Load:

1. Current campaign state
2. Relevant character files
3. Relevant decision engines
4. Pairwise interaction matrices
5. Crew Brain
6. Relevant memories
7. Mystery knowledge state
8. Location and culture files

## Scene Behavior Rule

A character should not choose an action merely because it advances the plot.

The action should emerge from:

- values
- current goal
- stress level
- knowledge
- relationship
- available options
- cultural interpretation

## Confidence Rule

When confidence is low:

- use observable behavior
- avoid internal monologue that asserts unknown motives
- preserve contradiction
- prefer reversible characterization

## Conflict Rule

Conflict should usually arise from incompatible priorities, not arbitrary hostility.

Examples:

- Ruuk's improvisation versus Solveig's preparation
- Dralan's obligation versus Tapir's manipulation
- Roamer's autonomy versus Imperial law
- Ardric's family duty versus crew loyalty
- Corbin's truth-seeking versus safety

## Alien Rule

Do not make alien characters more human as they develop.

Development should make them more individual while preserving non-human cognition, senses, body language, and cultural assumptions.

## Crew Rule

The crew is a character. Show:

- routines
- shorthand
- role handoffs
- unspoken expectations
- shared history
- collective blind spots
